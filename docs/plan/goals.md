# Image to Splat の目標

作成日: 2026-06-10 23:47
更新日: 2026-06-10 23:47

このページは Image to Splat が目指す状態をまとめる。
初期の大きな計画は `docs/plan/plan.md` に残し、このページは現在の開発目標として更新していく。

## 大きな目標

Image to Splat は、1 枚の画像から 3D Gaussian Splatting を生成し、その結果をビューアで確認・調整・エクスポートできる、研究用途のスタンドアロンデスクトップアプリを目指す。

[TripoSplat](https://github.com/VAST-AI-Research/TripoSplat)（VAST AI Research, MIT）による推論をアプリに内蔵し、ComfyUI を経由せずに「画像入力 → 生成 → 3D 表示 → パラメータ調整 → 書き出し」を一つのアプリで完結させる。

## 目指す利用体験

- 画像をドラッグ&ドロップ、またはダイアログで読み込める。
- 生成パラメータ（ガウシアン数の上限など）を指定して TripoSplat 推論を実行できる。
- 生成された 3D Gaussian をビューポートでオービットカメラ操作しながら確認できる。
- 表示パラメータ（点スケール、露出、背景、ソート品質など）をリアルタイムに調整できる。
- 生成結果を `.ply` / `.splat` 形式でエクスポートできる。
- 生成の進捗・GPU 状態・ログをアプリ内で確認できる。

## 主要機能の目標

| 領域 | 目標 |
|---|---|
| 画像入力 | ドラッグ&ドロップ / ファイルダイアログで画像を読み込む |
| 生成 | 内蔵 TripoSplat（Python サイドカー）で画像から Gaussian を生成する |
| ビューア | Three.js + @mkkellogg/gaussian-splats-3d で `.ply` / `.splat` を表示する |
| カメラ | オービット（回転・パン・ズーム）操作 |
| 生成パラメータ | ガウシアン数上限、シード、その他 TripoSplat 入力をUIから設定する |
| 表示パラメータ | 点スケール、露出/トーン、背景色、ソート品質などを調整する |
| エクスポート | `.ply` / `.splat` で書き出す |
| 実行環境 | venv 同梱の Python サイドカー、Blackwell GPU(CUDA 12.8/cu128)で動作 |
| ステータス | 生成進捗、GPU/CUDA 状態、サイドカーログを表示する |

## 設計方針

- 配布は当面考えず、研究用途の単一マシン運用を前提にする。容量やインストールの重さは許容する。
- 推論は Electron に溶かし込まず、独立した Python サイドカー（FastAPI）として spawn し、localhost HTTP で通信する。
- Python 環境は venv を同梱する。Blackwell(sm_120) 対応のため torch は `cu128` 以降を使う。
- モデル重みはインストーラに焼き込まず、初回取得（Hugging Face）でローカルに展開する。
- レンダラは Three.js + @mkkellogg/gaussian-splats-3d を採用し、ビューア側を薄く保って独自パラメータUIと密結合させる。
- UI は「画像 → 生成 → 表示 → 調整 → 書き出し」の流れに沿わせ、内部都合をそのまま見せない。
